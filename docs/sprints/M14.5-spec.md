```

BLOQUEIA:   nenhuma sprint Mobile direta
ESTIMATIVA: 5-6h
```

## 1. Objetivo

Entregar um **acompanhamento opt-in** do ciclo menstrual com tom
sóbrio: registro voluntário de início, sintomas e intensidade, e
visualização das fases em calendário 28-35 dias com cores. Sem
alertas, sem estatísticas competitivas, sem patologização. A feature
fica desativada por padrão e só aparece quando o usuário ativa
explicitamente em Settings (M15). Permite pular dias sem culpa.
Mensagem padrão em qualquer tela vazia: `"Registre quando quiser."`.

## 4. Restrições

- **Regra −1** (Anonimato): zero referência a IA, zero nomes reais
  hardcoded.
- Sem emojis em código, docs ou commits.
- Strings de UI em **Sentence case com acentuação completa PT-BR**.
- `accessibilityLabel` sem acento.
- Comentários em código `.ts`/`.tsx` sem acento.
- Mensagens de commit sem acento.
- TypeScript strict.
- Imports via alias `@/*`.
- **Tom sóbrio absoluto**:
  - Vocabulário evita `"controle"`, `"rastreamento"`, `"previsão"`.
    Usar `"Acompanhamento"`, `"Registro"`, `"Visualização das
    fases"`.
  - Sem alertas push, sem badge no menu, sem destaque "atrasou X
    dias". Calendário é leitura passiva.
  - Sem estatísticas comparativas (`"ciclo médio: 28 dias, melhor que
    a média"` está fora). Pode mostrar `"Duração do último ciclo: 29
    dias"` em muted, sem juízo.
  - Sem patologização. Sintomas são lista de chips opt-in, sem
    marcador `"problema"` ou `"normal"`.
  - Permite pular dias sem culpa: empty state em dias sem registro
    é cor de fundo neutra (sem outline alarmante), e o calendário
    nunca pinta o dia como "faltando".
- **Privacidade reforçada**:
  - Dados ficam **só no Vault local**, sem cache backend (não entra
    em `financas-cache.json` nem em qualquer cache compartilhado).
  - Pasta dedicada `inbox/saude/ciclo/` para isolar dos outros
    schemas mentais e financeiros.
  - Filtro de pessoa: feature ativável só para `pessoa_a`,
    `pessoa_b` ou ambas, configurável em Settings (M15).
- **Mensagens canônicas**:
  - Empty state geral: `"Registre quando quiser."`
  - Empty state após ativação: `"Pode registrar o início do
    primeiro ciclo quando quiser."`
  - Toast pós-Salvar: `"Anotado."` (igual aos outros formulários
    sóbrios).
- Não tocar em arquivos fechados de sprints anteriores.

## 5. Schema YAML completo

`inbox/saude/ciclo/YYYY-MM-DD.md`:

```yaml
---
tipo: ciclo_menstrual
data: 2026-04-29
autor: pessoa_a
data_inicio: 2026-04-12      # data de inicio do ciclo atual; null antes do primeiro registro
fase: lutea                  # folicular | ovulatoria | lutea | menstrual; auto via inferirFase
sintomas: [colica, fadiga]   # array opcional, vazio se nao houver
intensidade: 3               # 1-5, opcional
humor_associado: 3           # 1-5, opcional, para correlacao manual no desktop
texto: "noite tranquila."    # opcional
---
```

Todos os campos opcionais ficam como `null` no frontmatter quando
ausentes (não omitir a chave) — facilita parsing futuro pelo
backend.

## 11. Decisões tomadas

- **Fase inferida + override:** auto via heurística simples; radio
  no form permite override manual. Combinação default + autonomia.
- **Calendário 28/35 dias adaptativo:** 28 padrão; vira 35 se
  ciclo registrado dura > 28. Auto-detect, sem toggle manual.
- **Filtro pessoa em abas separadas:** preserva privacidade visual
  entre as duas pessoas. Cada uma vê só o próprio calendário.
- **Sem cache backend de ciclo:** dados ficam só no Vault local
  (privacidade reforçada, ADR-0007). Cache opcional fica como
  decisão futura via toggle adicional em Settings se demanda
  surgir; não é "v2", é decisão de privacidade.
