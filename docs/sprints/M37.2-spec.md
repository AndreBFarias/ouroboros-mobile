## 1. Objetivo

Adicionar escrita ao Google Calendar pela rota `/agenda`: FAB
"Novo evento" abre `<SheetNovoEvento>` com form, persiste no
Calendar via API, atualiza cache local. Suporte também para
deletar evento via long-press na lista do dia. Exige re-autenticação
para subir o escopo de `calendar.events.readonly` (M37.1) para
`calendar.events` (read+write).

## 4. Restrições

- **Regra −1**: zero IA, zero nomes reais hardcoded.
- Sentence case + acentuação PT-BR.
- TS strict.
- **Sem escrita offline** (decisão M37.2 §9): se sem rede, FAB
  Novo evento fica desabilitado com tooltip "Sem conexão". Sem
  fila local — evita conflitos com sync nativo do Google.
- **A18 mantida** no `<SheetNovoEvento>`: precisa ter `<Screen>`
  por trás (não só `<BottomSheet>` solto).
- **Idempotência**: criar evento 2x com mesmo input em janela de
  60s não duplica (verifica via cache local + request_id).
- **Escopo escalado requer reconsentimento**: usuário precisa
  passar pelo browser de novo. Não tentar truque de "upgrade
  silencioso".

## 9. Decisões tomadas

- **Sem escrita offline**: evita complexidade de queue local e
  conflitos com Google sync nativo. Se sem rede, FAB desabilitado
  com tooltip; usuário cria depois.
- **Reconsentimento para subir escopo readonly → write**: Google
  obriga; UX explícita via banner em vez de loop silencioso.
- **FAB "Novo evento" verde no canto direito**: não conflita com
  `<FABMenu>` purple esquerda (z-index 10 vs 11 vide CONTRACT
  §7.10). Mesmo padrão de M34 (`<MenuCapturaVerde>` em /memoria).
- **Long-press para deletar**: evita ação acidental (vs swipe).
  Confirm sheet obrigatório.
- **Atualização otimista do cache**: cache atualiza imediato após
  POST 200, sem esperar próximo `listarEventos`. Reconcilia em 1h
  via TTL.
- **Idempotência via cache local + request_id**: evita criar 2
  eventos se rede flaky causar retry.
- **Timezone fixo `America/Sao_Paulo`**: usuário está no Brasil;
  evita bug de fuso. Sprint futura pode pegar do device.
- **Escopo mínimo `calendar.events`** (não `calendar` full): só o
  que o app precisa. Não pede acesso a calendários secundários nem
  metadados.
