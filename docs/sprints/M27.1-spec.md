# Sprint M27.1 — Fix boot screen lento e sobreposto à Home

```
DEPENDE:    M27 (overlays globais e _layout.tsx atualizado)
            M25 (OuroborosLoader como boot screen)
            M28 (achado COLAT-01 sobre useFonts oscilante)
BLOQUEIA:   nenhuma (UX correction)
ESTIMATIVA: 1-2h
```

## 1. Objetivo

Corrigir bug detectado em validação manual (2026-05-03,
`docs/validacao-manual-2026-05-03/RELATORIO.md` Achado #2): ao
recarregar `/` o boot screen `<OuroborosLoader />` aparece e demora
**~10s** até a Home montar. Mesmo após Home aparecer no DOM
(`bodyText` mostra "Hoje / A / Humor do dia / Carregando..."),
screenshots fullscreen mostram o boot screen sobreposto à Home —
sem z-index alto detectável (apenas FABMenu z=10).

Combina com achado COLAT-01 do executor M28: `useFonts` em SDK 54
oscila `loaded=true/false`, causando re-mount do early-return com
loader. `useConquistas` em `/calendario` também trava em web.

## 2. Hipóteses a investigar

1. **`useFonts` oscilante**: em `app/_layout.tsx`, o gate
   `if (!loaded) return <bootscreen />` re-monta cada vez que
   `loaded` flicka. SDK 54 web pode ter bug com
   `@expo-google-fonts/jetbrains-mono`.
2. **BiometriaGate sem cleanup**: timing de
   `setIsAuthenticated(true)` pode disparar re-render que monta
   loader novamente.
3. **useConquistas Promise nunca resolve**: `lerConquistas(vaultRoot)`
   com `vaultRoot` mock (`web://...`) fica preso em `loading=true`.
   Adapter web do FileSystem não retorna.
4. **Múltiplos gates em paralelo**: 3 stores hidratam
   independentemente (`useOnboarding`, `useVault`, `useSessao`).
   Se qualquer um demora, boot espera.

## 3. Entregáveis (após investigação confirmar raiz)

### Caminho A — Se for `useFonts` oscilante

- `app/_layout.tsx` — adicionar guard `useRef` para evitar re-mount
  do boot screen após primeira hidratação:
  ```tsx
  const jaCarregouFontes = useRef(false);
  useEffect(() => { if (loaded) jaCarregouFontes.current = true; }, [loaded]);
  const mostrarBoot = !loaded && !jaCarregouFontes.current;
  if (mostrarBoot) return <bootscreen />;
  ```

### Caminho B — Se for BiometriaGate timing

- `src/components/auth/BiometriaGate.tsx` — adicionar
  `useEffect` com cleanup que cancela check pendente no unmount.
- Considerar trocar para Suspense + boundary em vez de gate
  imperativo.

### Caminho C — Se for useConquistas

- `src/lib/conquistas/leitura.ts` — adicionar fallback web:
  ```ts
  if (vaultRoot.startsWith('web://')) {
    return [];  // mock vazio em web
  }
  ```

### Caminho D — Loader sobreposto após Home

- `app/_layout.tsx` — usar `<Animated.View>` com fade out do
  loader em vez de unmount imediato:
  ```tsx
  return (
    <>
      <Stack>...</Stack>
      {!totalmenteCarregado && (
        <FadingLoader />  // fade out 200ms quando totalmenteCarregado vira true
      )}
    </>
  );
  ```

### Arquivos de teste

- `tests/app/_layout.test.tsx` — se existir, adicionar caso para
  boot guard. Se não existir, considerar criar (mas evitar testar
  side-effect de boot — preferir teste unitário do helper extraído).
