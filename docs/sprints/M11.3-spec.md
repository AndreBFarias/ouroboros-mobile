# Sprint M11.3 — Grid Fotos respeita largura do frame mobile

```
DEPENDE:    M11.1 fechada
BLOQUEIA:   nada (cosmético)
ESTIMATIVA: 0,5h
PRIORIDADE: baixa (afeta apenas Gauntlet/Web; mobile real OK)
STATUS:     [todo]
```

## 1. Achado / motivação

Validação visual M34 via Gauntlet
(`docs/sprints/M34-screenshots-gauntlet/A-foto-na-galeria.png`)
revelou que ao adicionar **1 foto** via `__gauntlet.adicionarFotoMock()`,
o thumbnail renderiza ocupando **largura quase total** (~ 380px),
não como célula de grid 3-colunas dentro do frame 412dp esperado.

Causa raiz: `MemoriasFotosTab.tsx:46-50` calcula `thumbSize` via
`useWindowDimensions().width` — em web isso retorna a largura do
**viewport** (1280px), não a do frame mobile (412dp aplicado em
`FrameMobileGauntlet`). Resultado: thumbSize ≈ 400dp e cada célula
"vaza" para fora do frame.

```ts
const thumbSize = useMemo(() => {
  const padding = spacing.lg * 2;
  const gaps = spacing.sm * (COLS - 1);
  return Math.floor((dim.width - padding - gaps) / COLS);
}, [dim.width]);
```

Em mobile real, `dim.width` = 412 → thumb ~124dp (correto). Em web
Gauntlet, `dim.width` = 1280 → thumb ~400dp (errado).

## 2. Caminho

Substituir `useWindowDimensions().width` por constante 412dp **em
modo Gauntlet** (web + dev), mantendo dinâmico em release mobile:

```ts
import { Platform } from 'react-native';
import { GAUNTLET_ATIVO } from '@/lib/dev/gauntlet';

const FRAME_W = 412;
const larguraEfetiva = GAUNTLET_ATIVO ? FRAME_W : dim.width;
const thumbSize = Math.floor((larguraEfetiva - padding - gaps) / COLS);
```

Aplicar mesma lógica a outros componentes que usam
`useWindowDimensions` em renders dependentes do frame
(`HeatmapBase`, `Calendar`, etc. — auditar com grep).

## 3. Entregáveis

- Fix em `MemoriasFotosTab.tsx` thumbSize.
- Helper compartilhado `src/lib/ui/larguraFrame.ts` exportando
  `useLarguraFrame()` que centraliza a lógica.
- Auditar com `grep useWindowDimensions src/` outros consumidores;
  migrar para o hook quando aplicável.
- Screenshot novo em `docs/sprints/M11.3-screenshots-gauntlet/A-grid-fotos-3-cols.png`.

## 4. Verificação

- `seedComDados('humores-30d')` + `adicionarFotoMock` × 4 → grid
  exibe 3 colunas + 1 célula na 2ª linha + 2 placeholders.
- Largura de cada thumb ~124dp em frame 412.

## 5. Decisões tomadas

- Constante `FRAME_W = 412` é mesma do `FrameMobileGauntlet`. Se um
  dia mudar (M-FRAME-IPAD futuro), centraliza num único token.
- Hook em vez de função pura — consumir via React garante reatividade
  se o frame ganhar variação dinâmica futuramente.
