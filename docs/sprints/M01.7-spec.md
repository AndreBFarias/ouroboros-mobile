## 1. Achado / motivação

`src/components/ui/Button.tsx` deriva `accessibilityLabel` direto do
`label` (string). Quando consumidor precisar passar um `label` JSX
(ex: ícone + texto), a derivação falha porque ReactNode não é
string.

Achado documentado em M11.1 ao precisar de botão com microformatação
no empty state.

## 2. Caminho

`<Button>` aceita prop `accessibilityLabel?: string` opcional:

```tsx
const a11yLabel =
  accessibilityLabel ?? (typeof label === 'string' ? label : undefined);
```

E `label?: string | ReactNode`. Render condicional:

```tsx
{typeof label === 'string' ? <Text>{label}</Text> : label}
```

## 3. Entregáveis

- Refactor em `src/components/ui/Button.tsx`.
- 3 cases novos em `tests/components/ui/Button.test.tsx`:
  - `label="Salvar"` → `accessibilityLabel="Salvar"` (compat).
  - `label="Salvar" accessibilityLabel="botao salvar"` → override.
  - `label={<Text>Foo</Text>} accessibilityLabel="botao foo"` →
    visual JSX + a11y string.
- Auditoria via `grep -rn "<Button" src/ app/` confirma zero
  consumidor existente quebra.

## 4. Verificação

## 5. Decisões tomadas

- Tipo `string | ReactNode` é superset compatível: chamadas

- Sem mudança no DOM/visual quando `label` é string (compat 100%).
