
## 1. Objetivo

Entregar uma tela nova (Tela 25, fora do conjunto original de 22)
dedicada a celebrar as conquistas registradas pelo usuário. Cada
conquista vira card num timeline horizontal scrollável com cover
visual extraído da mídia anexada (foto, thumbnail YouTube, album
art Spotify ou waveform de áudio). Tap no card abre uma sub-tela
de detalhe. Filtros por pessoa, mês e tipo de mídia. A tela só faz
sentido depois que M07.x estiver implementado (mídia obrigatória
em conquistas), porque sem mídia o cover ficaria genérico e perderia
o tom de "fechamento emocional".

## 4. Restrições

- **Regra −1** (Anonimato): zero referência a IA, zero nomes reais.
  Detalhes em `CLAUDE.md` e `VALIDATOR_BRIEF.md` §1.1.
- **Bloqueio explícito de M07.x**: esta sprint **não pode iniciar
  antes de M07.x** estar implementado. M07.x adiciona o campo
  `midia` aos schemas `evento` e `diario_emocional` com refinement
  condicional. Se M07.x não estiver pronto, o loader desta sprint
  retorna sempre array vazio (porque `midia.length > 0` nunca seria
  satisfeito), e a tela ficaria permanentemente em empty state.
- **Sem gamificação** (ADR-0005): cards são fechamento emocional,
  não troféus. Zero ranking, zero pontos, zero comparativo entre
  pessoas.
- Sem emojis em código, docs ou commits.
- Strings de UI em **Sentence case com acentuação completa PT-BR**.
  Exemplos canônicos: `"Calendário"`, `"Conquistas"`,
  `"Filtrar por mês"`, `"Filtrar por mídia"`, `"Foto"`, `"YouTube"`,
  `"Spotify"`, `"Áudio"`,
  `"Sua primeira conquista vai aparecer aqui."`,
  `"Tudo"`, `"Este mês"`.
- `accessibilityLabel` sem acento.
- Comentários em código `.ts`/`.tsx` sem acento.
- Mensagens de commit sem acento.
- TypeScript strict — sem `any`.
- Imports via alias `@/*`.
- Cover de YouTube usa **apenas a thumbnail HTTPS pública**
  (`https://img.youtube.com/vi/{id}/hqdefault.jpg`); não embutir
  player de vídeo.
- Cover de Spotify pode ser placeholder com ícone se a API não
  estiver disponível. Decisão final em "Dúvidas em aberto".

## 9. Dúvidas em aberto

- **Spotify cover:** usar Web API com PKCE flow ou só link colado +
  placeholder? Web API requer credenciais que entram em SecureStore;
  trade-off entre UX (cover automático bonito) vs complexidade
  (auth flow + refresh token + rede). Default proposto: placeholder
  com ícone na M11.5; integração API em sprint futura M11.5.1.
- **Filtros adicionais:** filtro por intensidade do evento
  (slider 1-5)? Filtro por bairro? Hoje a spec assume 3 filtros:
  pessoa, mês, tipo de mídia.
- **Stagger de animação:** com 30+ cards, o stagger 50ms vira 1,5s.
  Limitar stagger total a 600ms?
- **Detalhe da conquista:** abrir modal full screen, sub-rota stack,
  ou bottom sheet? Hoje a spec assume sub-rota stack
  (`/calendario/[id]`).
- Tela 25 entra ou não no `docs/Ouroboros_22_telas-standalone.html`
  como artboard novo? Hoje o mockup tem 22; precisa decisão para
  manter consistência da fonte de verdade visual.
