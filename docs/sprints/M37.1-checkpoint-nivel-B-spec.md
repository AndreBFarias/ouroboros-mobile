## 1. Motivação

M37.1 entregou 5 screenshots Nível A via **mock OAuth** no
Gauntlet (web + `__DEV__`). OAuth real (redirect
`ouroboros://oauth-callback` em custom-scheme) **não funciona em
browser** — exige runtime nativo. Esta sprint cobre o checkpoint
diferido para fechar M37.1 release-ready.

## 2. Pré-requisitos (dono prepara antes do dispatch)

- **APK dev-client fresh** em
  `/home/andrefarias/Desenvolvimento/Protocolo-Mob-Ouroboros/builds/dev-client-*.apk`
  (datado >= 2026-05-05; o de 2026-05-02 não tem deps M37.1
  como `expo-auth-session`/`react-native-calendars`).
- **Test user cadastrado no Google Cloud Console** em
  `OAuth consent screen → Test users` (qualquer conta Gmail
  serve enquanto o consent screen estiver em modo `Testing`).
- **Emulador configurado** (`./scripts/start-emulator.sh` deve
  subir `ouroboros-test` sem erro — já validado em sprints
  anteriores).

## 3. Tarefa (executor segue passo a passo)

```bash
cd /home/andrefarias/Desenvolvimento/Protocolo-Mob-Ouroboros

# 1. subir emulador
./scripts/start-emulator.sh
adb devices  # confirmar 'ouroboros-test  device'

# 2. instalar APK
adb -s ouroboros-test install -r builds/dev-client-*.apk

# 3. abrir app
adb -s ouroboros-test shell am start -n com.ouroboros.mobile/.MainActivity
```

Em seguida, navegar manualmente:

1. App abre na rota raiz; abrir MenuLateral → "Agenda".
2. Em `/agenda`, tap em **"Conectar conta Google"**.
3. WebBrowser abre prompt Google login real.
4. Logar com test user, aceitar escopo
   `https://www.googleapis.com/auth/calendar.events.readonly`.
5. Voltar ao app via redirect; agenda preenche com eventos reais
   da conta.
6. Confirmar arquivo persistido:
   ```bash
   adb -s ouroboros-test shell run-as com.ouroboros.mobile cat \
     /sdcard/Documents/Ouroboros/media/cache/agenda-pessoa_a.json | head -50
   ```
7. Capturar **3 screenshots** em
   `/home/andrefarias/Desenvolvimento/Protocolo-Mob-Ouroboros/docs/sprints/M37.1-screenshots/`:
   - `B-google-login-prompt.png` (prompt OAuth)
   - `B-agenda-eventos-reais.png` (lista populada)
   - `B-banner-invalido-real.png` (após revogar acesso em
     `https://myaccount.google.com/permissions` e reabrir app —
     banner "Sua conexão Google expirou.")

```bash
adb -s ouroboros-test exec-out screencap -p > \
  docs/sprints/M37.1-screenshots/B-<nome>.png
```

## 7. Decisões tomadas

- **Test user em vez de OAuth consent verificado**: M37.1 está
  em modo `Testing` no Google Cloud (escopo sensível
  `calendar.events.readonly`). Verificar consent só faz sentido
  pré-publicação na Play Store, fora do escopo MVP v1.0.
- **3 screenshots em vez de 2**: o spec original M37.1 §8 pedia
  só `B-google-login-prompt.png` e `B-agenda-eventos-reais.png`.
  Adicionamos `B-banner-invalido-real.png` para validar o caminho
  de erro `marcarInvalido` (revogação externa) em runtime real,
  que o mock Nível A não reproduz fielmente.
- **`run-as` em vez de `cat` direto**: `/sdcard/Documents/...`
  exige permissão de FileSystem; `run-as` com applicationId
  resolve em emulador `userdebug`.
