## A4. Filtros — reconciliar inconsistência do spec

Inconsistência detectada entre seções:

- Seção 2 (Entregáveis) menciona **3 filtros**: pessoa, mês, tipo de
  mídia. `filtros.ts` com 3 funções puras.
- Seção 9 (Definição de Pronto) menciona **5 filtros**: pessoa, mês,
  tipo de mídia, **intensidade (slider 1-5)**, **bairro (input
  livre)**.
- Seção 10 (Decisões) confirma os 5 filtros como decisão final.

**Resolução**: vale a Seção 10. M11.5 entrega **5 filtros**:

1. `filtrarPorPessoa(conquistas, pessoa)` — pessoa_a / pessoa_b /
   ambos.
2. `filtrarPorMes(conquistas, mes)` — `'este_mes' | 'mes_passado' |
   'tudo' | { ano, mes }`.
3. `filtrarPorTipoMidia(conquistas, tipo)` — `'tudo' | 'foto' |
   'youtube' | 'spotify' | 'audio'`.
4. `filtrarPorIntensidade(conquistas, [min, max])` — slider 1-5.
   Lê `meta.intensidade` (campo já presente em `EventoMeta` e
   `DiarioEmocionalMeta`; verificar via grep antes de implementar).
5. `filtrarPorBairro(conquistas, query)` — match case-insensitive
   contra `meta.lugar.bairro` (eventos) ou `meta.lugar` (diário).
   Input livre, debounce 300ms na UI.

Atualizar testes esperados em A1: `filtros.test.ts` cresce para
~16-20 casos (5 filtros × ~3 cenários + combinações).

## A5. Tratamento de mídia órfã (degradação graciosa)

**Risco conhecido** (registrado pelo validador M07.x): M11.5 lê
`meta.midia[].path` de eventos/diário. Arquivos órfãos no Vault podem
gerar links quebrados quando `MidiaFotoTab` ou audio recorder gravar
metadata sem persistir o arquivo final. Sub-sprint
`INFRA-midia-vault-orphan-reaper` está aberta para tratar a raiz, mas
M11.5 **não pode esperar** — deve degradar graciosamente.

**Política M11.5**:

- `<CoverMidia tipo="foto" uri={...}>` — usar `<Image>` com
  `onError` que troca para placeholder `bg-elev` + ícone `ImageOff`
  cor `--muted`. Não logar erro (offline / arquivo deletado é
  esperado).
- `<CoverMidia tipo="audio" uri={...}>` — `<WaveformPreview>`
  decorativo já não toca o arquivo (visual puro). Hash determinístico
  do path gera barras mesmo sem arquivo. Sem mudança.
- `<DetalheConquista>` mídia áudio — verificar
  `FileSystem.getInfoAsync(uri).exists` antes de montar player. Se
  false, mostrar mensagem em sentence case PT-BR: `"Áudio não
  disponível neste dispositivo."` e ícone `FileX` cor `--muted`.
  **Não remover** a conquista da lista (pode estar pendente de sync
  via Syncthing).
- Loader (`src/lib/conquistas/loader.ts`) **não filtra órfãos** —
  apenas reporta. Filtragem por existência atrasaria a UI e violaria
  princípio de leitura otimista do Vault. Adicionar comentário em
  PT-BR explicando a decisão.

Adicionar teste em `loader.test.ts`:

```ts
it('mantem conquista mesmo quando arquivo de midia esta ausente', async () => {
  // mock FS retorna false para getInfoAsync; loader deve incluir.
});
```

Conteúdo novo: rota real renderizando `<CalendarioConquistasScreen />`.

spec original (Seção 2) está incorreto sobre isso. **Não tocar
`_layout.tsx`**; o roteamento condicional é feito por `BottomTabs.tsx`
via `toggleByName`.
