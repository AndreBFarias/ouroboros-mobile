## Ressalvas (Tratadas em Sprint M01.6)

### R1. Regra de capitalização revogada

O `BRIEFING.md` §1 e §2.4 prescreviam **lowercase intencional** em
toda a UI. **O usuário (dono do projeto) revogou esta regra durante o
checkpoint visual M01.5.** A nova regra é:

- Strings de UI em **Sentence case** (primeira letra maiúscula, resto
  conforme uso natural PT-BR).
- **Acentuação completa PT-BR obrigatória** em toda string visível.
- `accessibilityLabel` continua sem acento (convenção de screen
  reader).
- Comentários em código continuam sem acento (convenção shell/CI).

Ver `VALIDATOR_BRIEF.md` §1.4 atualizado.

### R2. Acentos faltando em várias strings

Strings observadas sem acento que devem ganhar acento:

- `botoes` para `Botões`
- `padrao` para `Padrão`
- `anotacao` para `Anotação`
- `diario` para `Diário`
- `o que voce sentiu hoje.` para `O que você sentiu hoje.`
- `irritacao` para `Irritação`
- `cansaco` para `Cansaço`
- `notificacoes` para `Notificações`
- `ansiedade`, `serenidade`, `casa`, `trabalho`, `lazer` — já corretos

### R3. Densidade visual alta

Conteúdo agrupado, falta respiração entre seções e elementos.
Aplicar:

- Gap entre `<Section>`: 24 para **40dp**
- Espaço acima do título de seção: 16 para **24dp**
- Gap interno entre items dentro de seção: 8 para **12-16dp**
- Padding lateral da Screen: 20 (ok), mas usar `pt-6` no topo da tela
- Line-height base nos labels: 1,5 para **1,6** em body

### R4. Warning de SafeAreaView (ruído)

Console mostra:
```
WARN  SafeAreaView has been deprecated and will be removed in a future
release. Please use 'react-native-safe-area-context' instead.
```

Vem de dependência transitiva (`@gluestack-ui/themed`). Não é nosso
código direto. Suprimir via `LogBox.ignoreLogs([...])` em
`app/_layout.tsx` para reduzir poluição de log durante dev.

### M01.6.3 — Sobreposição Vitória/Trigger (commit pendente)

Screenshot [`06-fab-radial-sobreposicao-detectada.jpg`](./M01.5-screenshots/06-fab-radial-sobreposicao-detectada.jpg)
revelou que com labels maiores e fundo sólido, os botões "Vitória"
(252°) e "Trigger" (270°) ficavam em ângulos próximos demais e seus
labels se sobrepunham.

Solução aplicada:
- Espaçamento angular entre itens: 18° para **22°**.
- Ângulos novos: Humor 175°, Voz 197°, Câmera 219°, Exercício 241°,
  Vitória 263°, Trigger 285°.
- ARC_RADIUS: 150 para **175** (mais espaço linear entre labels).
