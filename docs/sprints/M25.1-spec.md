# Sprint M25.1 — Fix animação OuroborosLoader em web (rotação em torno do centro)


## 1. Objetivo

Corrigir bug detectado em validação manual (2026-05-03,
`docs/validacao-manual-2026-05-03/RELATORIO.md` Achado #1): a
animação de rotação dos 4 anéis do `OuroborosLoader` não funciona
visualmente em web. O Reanimated atualiza os valores das shared
values, mas `react-native-svg-web` converte
`<G rotation={N} originX={160} originY={160}>` para `<g
transform="rotate(N)">` **sem cx/cy** — a rotação acontece em torno
de (0,0) ao invés do centro (160, 160), jogando o conteúdo pra fora
do viewBox visível.

Adicionalmente, no boot screen 320px, o atributo `rotation` fica
preso em `"0"` permanente — `useAnimatedProps` não chega a propagar
porque o boot termina antes do primeiro frame de animação.

## 2. Entregáveis

### Arquivos modificados

- `/home/andrefarias/Desenvolvimento/Protocolo-Mob-Ouroboros/src/components/brand/OuroborosLoader.tsx`

  react-native-svg-web.

  Bundle Hermes ±0 KB.
