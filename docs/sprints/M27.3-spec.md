# Sprint M27.3 — Boot screen sem oscilar via Suspense boundary

```
DEPENDE:    M27.1 fechada (useRef guard)
            + M27.2 deferido (analise documentada)
BLOQUEIA:   nenhuma (M27.1 cobre 95% dos casos)
ESTIMATIVA: 4-6h
PRIORIDADE: media (UX residual em sessão fresh do Chrome dev)
STATUS:     [todo]
```

## 1. Contexto

M27.1 (useRef guard) e M27.2 (tentativa de latch global) deixaram
o seguinte estado:

- Em `/_dev/gauntlet`: 0 oscilação observável.
- Em rotas reais (`/`, `/humor`, `/settings`, `/memoria`): pode
  oscilar entre boot screen e conteúdo na primeira sessão fresh
  porque `useFonts` SDK 54 web demora 30-60s e há outros gates
  (BiometriaGate, SessaoBootGate) que disparam re-renders.
- Tentativas de fix com `let` de módulo ou `sessionStorage` durante
  render causaram `Maximum update depth exceeded` em React 19
  strict mode.

## 2. Solução proposta

Refatoração via **Suspense boundary** orquestrado:

1. `app/_layout.tsx` envolve children num `<Suspense fallback={<BootScreen />}>`.
2. Um hook `useAppPronto(): boolean` agrega:
   - `useFonts` resolvido E
   - `useHasHydrated(useOnboarding)` E
   - `useHasHydrated(useVault)` E
   - `useHasHydrated(useSessao)` E
   - (opcional) `useHasHydrated(useNavegacao)`.
3. Enquanto `useAppPronto() === false`, hook lança Promise pendente
   (`throw new Promise(resolve => onPronto.subscribe(resolve))`)
   ativando Suspense fallback.
4. `expo-splash-screen` continua escondido até Suspense resolver
   primeira vez.
5. Após primeira resolução, hook retorna `true` permanentemente
   (latch em ref OU em store leve `useBootStatus` com selector
   estável).

## 3. Pontos de atencao

- **React 19 strict mode**: ref + setState normal funciona, `let`
  de módulo NÃO. Nunca mexer em var de módulo durante render.
- **expo-splash-screen `hideAsync`**: chamar exatamente UMA vez.
- **Suspense boundary em RootLayout**: tem que ficar dentro do
  `GestureHandlerRootView`, mas FORA do `BiometriaGate`. Senão
  bypass do Gauntlet quebra.
- **Não introduzir `Suspense` em React 18.x do Android nativo** se
  o Reanimated worklet não suporta — verificar pacote.

## 4. Entregáveis

- `src/lib/boot/useAppPronto.ts` (hook agregador).
- `src/lib/boot/useBootStatus.ts` (store zustand leve, sem persist).
- `app/_layout.tsx` envolto em `<Suspense>` na raiz.
- `tests/lib/boot/useAppPronto.test.tsx` (mocks de hidratação).
- `tests/e2e/playwright/m27-3-boot-suspense.e2e.ts`:
  - Reload em `/`, contar transições do `[aria-label="loader ouroboros"]`.
  - Esperado: 1 transição (presente → ausente). Nunca volta.
