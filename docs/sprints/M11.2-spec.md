## 1. Achado / motivação

`MemoriasFotosTab.tsx` (M11.1) e `useFotosAgregadas.ts` leem
`useGaleriaMock` no corpo do componente/hook a cada render. O store
é dev-only e protegido por `GAUNTLET_ATIVO`, mas a leitura síncrona
no corpo:

1. Adiciona deps de re-render desnecessárias quando o store muda em
   web/dev.
2. Aciona o caminho mock fora de `useEffect` — não é colateral, mas
   viola a convenção CONTRACT §7.9 (efeitos visuais devem nascer em
   `useEffect`, não em render direto).

## 2. Caminho

Mover leitura para `useEffect` guardado por `GAUNTLET_ATIVO`:

```ts
const [fotosMock, setFotosMock] = useState<FotoMock[]>([]);

useEffect(() => {
  if (!GAUNTLET_ATIVO) return;
  const unsub = useGaleriaMock.subscribe((s) => setFotosMock(s.fotos));
  setFotosMock(useGaleriaMock.getState().fotos);
  return unsub;
}, []);
```

Em release Android, `GAUNTLET_ATIVO === false` → effect retorna early →
mock store não é nem importado em produção (já garantido por
M-GAUNTLET-LEAK-CHECK).

  manter o branch dev isolado e tree-shake-friendly.
- Sem migração para Suspense — overkill para mock dev.
