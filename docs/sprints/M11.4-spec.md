# Sprint M11.4 — Evolução Corporal em Marcos (fotos perfil/lado + métricas)

```
DEPENDE:    M11 fechada (Marcos timeline com FAB unificado M34.3)
            + M12 fechada (MedidasSchema + galeria comparativa)
            + M34 fechada (MenuCapturaVerde)
BLOQUEIA:   M36 (Recap puxa "Evoluções" — quer agregação corporal)
ESTIMATIVA: 4-5h
```

## 1. Achado / motivação

Hoje:
- **Marcos** (M11) é timeline de conquistas pontuais (texto + tags
  + auto-marcos do backend). Não tem contexto corporal.
- **Medidas** (M12) é form mensal/semanal de medidas numéricas
  (peso, cintura, etc.) com 3 fotos comparativas (frente, lado,
  costas).

Estão **desconectadas**. Usuário quer registrar "tô na minha melhor
forma esse mês" tanto como marco emocional quanto como evidência
corporal — mas hoje precisa abrir 2 telas, em rotas diferentes,
com schemas diferentes.

## 2. Objetivo

Adicionar à aba **Marcos** uma **subseção "Evolução corporal"**
que mostra:

1. **Linha do tempo de medidas** (do `MedidasSchema` M12) com fotos
   miniaturas em cada ponto, ordem cronológica, tap abre detalhe
   M12 existente.
2. **Botão "Registrar evolução"** no FAB verde unificado da aba
   Marcos (M34.3 já fornece slot `acoesExtras`) que abre
   `app/medidas/novo` direto (sem precisar navegar via menu lateral).
3. **Vínculo opcional Marco→Medida**: ao criar marco manual, opção
   "Anexar medida deste período" lista as últimas 3 medidas e cria
   campo `marco.medidaRef` (slug em `medidas/`).

Sem schema novo — reusa `marco` (M11) e `medidas` (M12). Apenas
extensão do `MarcoSchema` com `medidaRef?: string` opcional.

## 4. Restrições

- **Regra −1** (anonimato).
- **Sentence case + acentuação PT-BR completa** em strings UI:
  "Evolução corporal", "Registrar evolução", "Anexar evolução
  corporal", "Última medida há 14 dias.", etc. Sem exceções.
- TS strict 0.
- Companion .md de cada foto (M34) preservado — fotos de medidas
  ficam em `media/fotos/medidas-<data>-{frente,lado,costas}.jpg`
  com `tipo: midia_foto` + `legenda: "Evolução corporal — frente"`.

## 6. Decisões tomadas

- **Não duplicar UI de medidas em Marcos**: reusa `/medidas/novo`
  via navigation. Mantém M12 como única fonte de verdade do schema.
- **`medidaRef` é slug curto** (e.g. `2026-05-04`), não path
  absoluto — facilita backup/restore.
- **Fotos de medidas seguem M39 companion** — quando M39 fechar,
  upgrade automático.
- **Recap (M36) consome essa subseção** via `useMedidas` agregado
  por período + correlaciona com `marcos[].medidaRef`.
