## Achados colaterais para sprints sucessoras

### Achado A — `M00.5.x-fix-fab-radial-labels`

**Identificado pelo usuário durante validação** ("os labels do radial
não tão maneiros também né"). FAB radial em viewport horizontal/web
mostra:

1. Labels com fundo `bgElev` em pill que parecem **tooltip improvisado**,
   não premium. Quebra a estética do app (que é minimalista).
2. Posicionamento misto: itens laterais (180-239°) com label à
   **esquerda** do círculo; itens superiores (240-270°) com label
   **acima**. Diferença de tratamento causa estranhamento visual.
3. Em viewport horizontal larga (desktop 1568px), labels podem
   estourar ou truncar (ex: "Conquista" parece "Conquist...").
4. Em viewport mobile (412dp), animação stagger não foi totalmente
   capturada visualmente — possível bug onde apenas o primeiro
   botão aparece, ou tempo de captura insuficiente.

**Sugestão de fix em sprint sucessora:**
- Labels embedded **dentro** dos círculos coloridos como subtítulo
  (ex: ícone Heart + texto "Humor" abaixo, mesmo botão).
- OU labels apenas em haptic + tooltip flutuante (sem fundo `bgElev`).
- OU revisar ângulos para distribuição mais homogênea.
- Avaliar consistência com mockup HTML standalone original.

Spec sucessora: `docs/sprints/M00.5.x-fix-fab-radial-labels-spec.md`
(a ser planejada via `/planejar-sprint` antes de fechar release).

### Achado B — `M19.x-bundle-html-toolchain`

Bundle `Ouroboros_22_telas-standalone.html` precisa de toolchain
local para regeneração. Hoje depende de `claude.ai/design`. Sub-sprint
da M19 deve:

1. Extrair script Node.js que faz bundle a partir dos JSX-fonte em
   `docs/design-canvas-export/project/secao-{a..e}.jsx`.
2. OU adotar Storybook como substituto da fonte de verdade visual.
3. Decidir numeração de Telas 25/26 (calendário/widget) vs t25-t28
   já existentes em `secao-e.jsx` (microfone/alarme/todo/contador).
