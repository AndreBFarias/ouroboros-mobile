## 1. Achado / motivação

M37.1 deixou `react-native-calendars` com locale default (en).
Header da rota `/agenda` mostra "May 2026" e dias da semana
"Sun Mon Tue Wed Thu Fri Sat". Conflita com a Regra de
Linguagem do `CLAUDE.md`: UI strings em PT-BR completo.

Validação visual de M37.1 capturou o comportamento explicitamente
(`A-agenda-mes-com-dots.png`), por isso a sub-sprint sai como
PT-BR audit-aware desde a primeira execução.

## 2. Tarefa

Registrar `LocaleConfig.locales['pt-BR']` em
`src/components/agenda/CalendarGrid.tsx` (ou em módulo
auxiliar `src/components/agenda/calendarLocalePtBr.ts` se for
mais limpo) com:

```ts
import { LocaleConfig } from 'react-native-calendars';

LocaleConfig.locales['pt-BR'] = {
  monthNames: [
    'Janeiro', 'Fevereiro', 'Março', 'Abril', 'Maio', 'Junho',
    'Julho', 'Agosto', 'Setembro', 'Outubro', 'Novembro', 'Dezembro',
  ],
  monthNamesShort: [
    'Jan', 'Fev', 'Mar', 'Abr', 'Mai', 'Jun',
    'Jul', 'Ago', 'Set', 'Out', 'Nov', 'Dez',
  ],
  dayNames: [
    'Domingo', 'Segunda-feira', 'Terça-feira', 'Quarta-feira',
    'Quinta-feira', 'Sexta-feira', 'Sábado',
  ],
  dayNamesShort: ['Dom', 'Seg', 'Ter', 'Qua', 'Qui', 'Sex', 'Sáb'],
  today: 'Hoje',
};
LocaleConfig.defaultLocale = 'pt-BR';
```

Idempotente — registro pode rodar em top-level do módulo
(executa uma vez no boot via require cache).

## 3. Restrições

- Anonimato Regra −1.
- **Acentuação completa obrigatória**: `Março`, `Sábado`,
  `Terça-feira`, etc. PT-BR audit hook bloqueia se faltar acento.
- TS strict 0 erros.
- Não introduzir nova dep — `LocaleConfig` é built-in do
  `react-native-calendars`.

## 8. Decisões tomadas

- Locale registrado em **module-top-level** (não via `useEffect`):
  é puro side-effect idempotente e não dependente de runtime
  context.
- Sem i18next ou react-intl — `LocaleConfig` cobre 100% do que
  precisamos.
- `dayNamesShort[6] = 'Sáb'` com acento para conformidade com
  PT-BR audit. Header do calendar exibe abreviação de 3 chars
  (cabe `Sáb`).
