# Sprint M34.1 — FABMenu sobrepõe sheets do MenuCapturaVerde

```
DEPENDE:    M34 fechada
BLOQUEIA:   nada
ESTIMATIVA: 1-2h
PRIORIDADE: média (UX desconforto, não funcional)
STATUS:     [todo]
```

## 1. Achado / motivação

Validação visual M34 via Gauntlet (2026-05-04, screenshot
`docs/sprints/M34-screenshots-gauntlet/A-sheet-frase.png`) revelou
que o **FABMenu roxo** (esquerda inferior, z-index 10 declarado em
`app/_layout.tsx` por CONTRACT §7.10) fica **flutuando sobre o
botão "Cancelar"** do SheetFrase — corta visualmente a faixa
inferior do sheet. O mesmo desconforto vale para qualquer sheet
que abre na tab Memórias (M11.1 FotoDetalhe + M34 SheetFrase + sheet
do menu verde).

Causa raiz: gorhom/bottom-sheet em web usa `position: absolute`
dentro do container do tab; o FABMenu é overlay global em
`_layout.tsx` raiz com z-index 10 e portanto cobre qualquer
sheet renderizado abaixo.

Comportamento esperado: ao abrir um BottomSheet, o FABMenu deveria
recuar ou ser ocultado (já existe `rotaEsconderFAB` em
`src/lib/navegacao/rotasSemFAB.ts` para casos análogos de rotas
modais).

## 2. Caminhos possíveis (escolher 1 na execução)

**Caminho A — sheet sobe z-index acima do FABMenu (preferido):**
Adicionar `containerStyle={{ zIndex: 30 }}` (ou contant `Z_SHEET = 30`)
no wrapper `BottomSheet` em `src/components/ui/BottomSheet.tsx` para
qualquer sheet ficar acima dos overlays globais (FABMenu z=10,
MenuLateral z=20).

**Caminho B — useNavegacao expõe flag `sheetAberto` e FABMenu
escuta:** Quando qualquer sheet abre, store seta `sheetAberto=true`
e FABMenu se esconde via `MotiView` com fade 200ms. Mais invasivo,
exige instrumentar 4-5 sheets.

**Caminho C — `gestureHandlerRootHOC` re-wrap:** ajustar contexto
gesto para o sheet "intercept" cliques no FABMenu. Não resolve
visual, só funcional.

Preferência: **A** (delta minimal, escopo isolado).

## 3. Entregáveis

- `src/components/ui/BottomSheet.tsx` ganha `containerStyle.zIndex: 30`
  como default (override por prop opcional).
- 1 teste novo em `tests/components/ui/BottomSheet.test.tsx` cobrindo
  z-index default 30.

## 4. Verificação

- Gauntlet: navegar `/memoria` → FAB verde → "capturar frase" →
  scroll do sheet até botão "Cancelar" → confirmar que FABMenu não
  sobrepõe.
- Screenshot novo em `docs/sprints/M34.1-screenshots-gauntlet/A-sheet-acima-fab.png`.

## 5. Decisões tomadas

- z-index 30 (não 100) para preservar margem de manobra futura.
- Default no componente, não em cada chamada — política de
  consistência (afeta 4 sheets atuais de uma vez).
