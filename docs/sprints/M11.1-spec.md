## 1. Achados visuais (validados via Gauntlet em 2026-05-04)

Ao navegar `/memoria` em uso real, 4 problemas estruturais aparecem:

1. **Aba Fotos passiva** — empty state "Suas fotos vão aparecer
   aqui conforme você registrar" sem botão para adicionar foto. A
   intenção original era que fotos chegassem automaticamente de
   M07.x (conquistas com mídia), M11 (marcos com foto), M13
   (exercícios com foto). Resultado: usuário vê aba inerte e sem
   ação possível.
2. **Aba Marcos funcional mas sem validação E2E** — FAB "+" abre
   BottomSheet "Novo marco" com campos Descrição, Tags (Treino /
   Consistência / Emocional / Vitória / Retomada), Salvar/Cancelar.
   Mas o save end-to-end nunca foi rodado pelo Gauntlet —
   provavelmente quebra em web por mock do Vault.
3. **Aba Treinos bloqueada por dependência** — FAB "+" abre form
   "Novo treino" (Rotina, Duração 30min slider, Observações,
   Salvar/Cancelar). Salvar fica `disabled` com mensagem "Cadastre
   exercícios na aba Galeria antes de montar um treino". A Galeria
   fica em outra rota (`/exercicios`). Sem atalho navegacional —
   usuário lê mensagem mas não tem ação imediata.
4. **Heatmap descentralizado** — grid 13×7 (91 dias) renderiza
   alinhado à esquerda enquanto header "Memórias" e empty states
   estão centralizados. Visualmente quebrado.

Screenshot do bug em `.playwright-mcp/memoria-fotos.png` (aba Fotos
sem FAB) e `.playwright-mcp/memoria-marcos.png` (aba Marcos com
FAB).

## 2. Entregáveis

### 2.1 Fotos com upload manual

Em `app/memoria.tsx` (ou no componente `FotosTab`):

- Adicionar FAB roxo "+" no canto inferior direito da aba Fotos
  (igual ao FAB de Marcos).
- `aria-label="adicionar foto"`.
- Ao tocar: abre `expo-image-picker` com `mediaTypes: 'images'`,
  `allowsMultipleSelection: false`, `quality: 0.8`.
- Após o usuário selecionar:
  - Helper canônico já usado em M13 / M11 marcos: copia o asset
    para `media/YYYY-MM-DD/IMG_<unix-ts>.jpg` no Vault.
  - Cria registro `MidiaSchema` (campo `tipo: 'foto'`, `origem:
    'galeria-manual'`, `data: hoje`, `path: <relativo ao vault>`).
  - Atualiza a galeria agregada (mesma store usada por M11) e a
    tela re-renderiza mostrando a foto na grid.
- Empty state ganha texto secundário: "Toque + para adicionar uma
  foto agora."

### 2.2 Atalho de Treinos para Galeria

Em `app/memoria.tsx` aba Treinos, dentro do empty state ou logo
acima do FAB:

- Botão de texto "Cadastrar exercícios na Galeria" que dispara
  `router.push('/exercicios')`.
- `aria-label="abrir galeria de exercicios"`.
- Mensagem do BottomSheet "Cadastre exercícios na aba Galeria antes
  de montar um treino" ganha link clicável para `/exercicios`.

### 2.3 Heatmap centralizado

Em `app/memoria.tsx` aba Treinos:

- Container do heatmap recebe `style={{ alignItems: 'center' }}`
  ou `paddingHorizontal: spacing.md` simétrico.
- Verificar que cada linha de 13 células fica horizontalmente
  centralizada no frame mobile 412dp.

### 2.4 Validação E2E de Marcos

Em `tests/e2e/playwright/m11-1-marcos-criar.e2e.ts`:

- `seed()` + abrir `/memoria` + clicar tab Marcos.
- Clicar FAB "+", preencher Descrição "Primeira semana cumprida",
  selecionar tag Consistência, clicar Salvar.
- Confirmar que o marco aparece na lista da aba Marcos.
- Capturar screenshots:
  - `docs/sprints/M11.1-screenshots-gauntlet/A-marcos-novo-form.png`
  - `docs/sprints/M11.1-screenshots-gauntlet/B-marcos-salvo.png`

### 2.5 Validação E2E de Fotos manuais

Em `tests/e2e/playwright/m11-1-fotos-upload.e2e.ts`:

- `seed()` + abrir `/memoria` + clicar tab Fotos.
- Confirmar que FAB "adicionar foto" está presente no DOM.
- Tentar acionar via `__gauntlet.adicionarFotoMock()` (helper a
  criar no Gauntlet — ver §2.6).
- Confirmar que a galeria recebe um item.

### 2.6 Helper Gauntlet `adicionarFotoMock`

Em `src/lib/dev/gauntlet.ts`:

- Adicionar método `adicionarFotoMock(): Promise<void>` na
  `GauntletAPI`.
- Em web mock, cria um registro `MidiaSchema` direto na store da
  galeria com `path: 'web://mock/foto-<ts>.jpg'`, sem invocar
  `expo-image-picker` (que falha em web).
- Em mobile real, este método é no-op (para evitar vazamento).

### 2.6 E2E unificado de regressão

`tests/e2e/playwright/m11-1-memorias-usavel.e2e.ts` cobre:

- Heatmap centralizado: `getBoundingClientRect()` da primeira
  linha tem `x` próximo a `(412 - largura_grid) / 2 ± 8px`.
- FAB Fotos presente.
- Botão "Cadastrar exercícios" presente na aba Treinos.

## 3. Restrições

- Sentence case + acentuação PT-BR completa em qualquer string
  nova de UI.
- Comentários sem acento.
- TypeScript strict.
- Não quebrar a galeria agregada existente (compatibilidade com
  fotos vindas de M07.x, M11 marcos, M13 exercícios).
- `expo-image-picker` é dev-client only no Android; em web o helper
  do Gauntlet faz mock determinístico (sem chamar a API real).

  +3-5 testes de integração da Fotos manuais e do helper Gauntlet.
- Smoke verde, tsc 0 erros, anonimato OK.
- Bundle Hermes Android: ≤ 8.85 MB (mudança é só JS).
- 3 E2E novos em `tests/e2e/playwright/`.
