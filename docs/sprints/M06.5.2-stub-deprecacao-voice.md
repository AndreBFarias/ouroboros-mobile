# Sprint M06.5.2 — Migracao Voice -> expo-speech-recognition (STUB)

> **Status:** STUB / proposta. Nao iniciada. Achado colateral durante
> execucao de M06.5 em 2026-05-01. Para virar spec executavel, rodar
> `/planejar-sprint` apontando para este arquivo como contexto.

## Contexto

Durante `npx expo install @react-native-voice/voice` na M06.5, o npm
emitiu:

```
npm warn deprecated @react-native-voice/voice@3.2.4: This package is
  deprecated. Use expo-speech-recognition instead.
```

A spec M06.5 (e seu adendo) exigiam `@react-native-voice/voice`
explicitamente, entao a M06.5 prosseguiu com a lib deprecated. Esta
sub-sprint avalia e (se viavel) executa a migracao para
`expo-speech-recognition`, que e a recomendacao oficial atual do
ecossistema Expo.

## Risco que mitiga

- Voice 3.2.4 nao recebe mais updates; pode quebrar em SDK 55+ do Expo.
- `expo-speech-recognition` e mantido pela equipe Expo; alinhamento
  futuro garantido.
- Ambas usam `SpeechRecognizer` Android nativo on-device (zero rede,
  ADR-0007 preservado).

## Touches autorizados (antecipados)

- `package.json` — remover `@react-native-voice/voice`, adicionar
  `expo-speech-recognition` via `npx expo install`.
- `app.json::plugins` — substituir bloco do voice pelo da nova lib
  (com strings PT-BR de permissao).
- `src/lib/diario/transcribe.ts` — re-implementar contra a nova API.
  Estrutura de listeners e shape de Promise devem permanecer
  compativeis com o caller (`MicrofoneButton.tsx` nao toca).
- `tests/components/diario/MicrofoneButton.test.tsx` — atualizar
  mock global no `jest.setup.cjs` para refletir nova lib.
- `jest.setup.cjs` — substituir mock de Voice pelo da nova lib.
- `CHANGELOG.md` — entrada Quality.

**Nao tocar:** `MicrofoneButton.tsx`, `recordAudio.ts`, `permissions.ts`,
`Waveform.tsx`. O contrato externo (callbacks `onTextoTranscrito` /
`onAudioGravado`) deve ficar identico.
