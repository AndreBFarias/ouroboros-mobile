instalado no celular `carsvg7du8kfnrlj` (SHA EAS `15da107f`,
`com.ouroboros.mobile v0.1.0`) **não inclui esses módulos** e vai
crashar no primeiro `import` em runtime.

3. `tsc --noEmit` passa.
4. `expo export --platform android` passa.

   `npm run build:dev`. Aguardar ~15-25 min na nuvem EAS.

7. Só então validar gravação real, transcrição e sync do `.m4a`.

## A4. Decisão de transcrição (clarificação)

Prompt do orquestrador questionou Whisper vs Vosk vs cloud. Spec já
fechou na Seção 10:

- **Voice nativo Android (`@react-native-voice/voice`)** — wrapper
  para `SpeechRecognizer` do sistema. **Zero peso adicional no APK**
  (~2 MB de glue code), zero rede (Android usa engine on-device a
  partir do Android 12+ se disponível; fallback Google App nativo).
- **Whisper.cpp / Vosk rejeitados nesta sprint** porque adicionam
  50-70 MB ao APK e exigem integração JNI complexa. Reservados para
  sub-sprint **M06.5.1** se o usuário relatar qualidade ruim do
  reconhecedor nativo na validação Nível C.
- **Cloud rejeitado** por `ADR-0007` (zero rede).

Esta decisão **mantém-se**. Adendo não altera.
