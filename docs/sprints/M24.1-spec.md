# Sprint M24.1 — Resume state não restaura ultimaRota

```
DEPENDE:    M24 fechada (resume state inicial)
            + M-REVALIDACAO-M20-M28 (achado registrado)
BLOQUEIA:   M29
ESTIMATIVA: 1-2h
PRIORIDADE: alta
```

## 1. Bug observado

Cenário gauntlet:
```js
__gauntlet.seed();
__gauntlet.setUltimaRota('/memoria');
location.assign('/');
```
Esperado: app abre em `/memoria`.
Atual: app abre em `/` (Home).

Evidência: `docs/validacao-gauntlet-2026-05-03/screenshots/M24/A-fail-redirect-home.png` + relatório.

## 2. Hipóteses (ordem de probabilidade)

1. **Reidratação da store assíncrona** — `useSessao` (zustand persist com SecureStore) só hidrata `ultimaRota` depois do primeiro render. `SessaoBootGate` lê `null` na primeira passada e não dispara `Redirect`.
2. **Condição quebrada** — pode existir `if (ultimaRota && ultimaRota !== '/' && pathname === '/')` que falha por race ou por comparação errada (ex: `pathname` no expo-router web vem com query string).
3. **Ordem entre BootGate e expo-router root** — Stack root navega antes do BootGate decidir.

## 3. Fix mínimo proposto

Em `src/lib/boot/sessaoBootGate.tsx` (ou `app/_layout.tsx` onde estiver o gate):
- Adicionar dependência da hidratação: `const _hasHydrated = useSessao.persist?.hasHydrated();`
- Só renderizar `Redirect href={ultimaRota}` se `_hasHydrated && pathname === '/' && ultimaRota && ultimaRota !== '/'`.
- Se já hidratou e nada a redirecionar, render Stack normal.
