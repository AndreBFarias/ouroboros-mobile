```
DEPENDE:    M06 fechada (Tela 18 com modo vitoria salvando em
              inbox/mente/diario/)
            + M07 fechada (Tela 20 com modo positivo salvando em
              eventos/)
            + M06.5 fechada (gravação de áudio, pré-requisito para
              tipo 'audio' do MidiaPicker)
BLOQUEIA:   M11.5 (Calendário visual de conquistas — depende deste
              schema com mídia para renderizar covers)
ESTIMATIVA: 5-7h
```

> **DEPENDÊNCIA CRÍTICA — TIPO `audio`.** O tipo de mídia `audio` do
> componente `<MidiaPicker>` reutiliza o pipeline de gravação criado
> na M06.5 (`src/lib/diario/recordAudio.ts`). Esta sprint **exige**
> que M06.5 esteja fechada. Sem versão reduzida — todas as 4 abas
> (Spotify / YouTube / Foto / Áudio) entregues juntas.

> **DECISÃO ARQUITETURAL SPOTIFY (FINAL).** Esta spec usa **oEmbed
> sem auth**: extrai `track_id` via regex, fetch único de
> `https://open.spotify.com/oembed?url=...` para metadata, sem
> persistir token. Quando oEmbed falha (offline, rede instável), o
> cover renderiza placeholder bonito com cor `--green` Dracula
> Spotify-like + ícone musical em `<View>`. Web API completa fica
> permanentemente rejeitada (contraria ADR-0007).

## 1. Objetivo

Tornar **mídia obrigatória** ao registrar conquistas. Quando o usuário
marca `modo: 'vitoria'` em `diario_emocional` (Tela 18) ou `modo:
'positivo'` em `evento` (Tela 20), o save **só** acontece se houver
pelo menos um item em um array `midia: []`. Tipos suportados: Spotify
(link), YouTube (link), Foto (galeria/câmera) e Áudio (gravação
on-device). Para `modo: 'trigger'` / `modo: 'negativo'`, mídia
permanece **opcional** — pedir mídia em momento de crise adiciona
fricção indevida (ADR-0005, sem gamificação).

A ideia: conquistas são marcos que merecem peso emocional duradouro;
forçar a captura de mídia evita registros vazios e dá material rico
para o calendário/timeline (M11.5).

## 4. Restrições

- **Regra −1** (Anonimato): zero referência a IA, zero nomes reais.
- Sem emojis em código, docs ou commits.
- Strings de UI em **Sentence case com acentuação completa PT-BR**
  (ADR-0013). Ex: `"Mídia"`, `"Spotify"`, `"YouTube"`,
  `"Foto"`, `"Áudio"`, `"Cole o link da música"`,
  `"Cole o link do vídeo"`, `"Escolher da galeria"`,
  `"Tirar foto"`, `"Gravar áudio"`, `"Adicionar"`,
  `"Adicione pelo menos uma mídia para conquista."`,
  `"Link de Spotify inválido."`, `"Link de YouTube inválido."`,
  `"Mídia adicionada."`.
- `accessibilityLabel` sem acento (ex: `'aba spotify'`,
  `'remover midia'`).
- Comentários em código sem acento.
- Mensagens de commit sem acento.
- TypeScript strict.
- Imports via alias `@/*`.
- **Não tocar** em `src/components/ui/*` (M01).
- **Spotify**: aceitar apenas link colado. **Sem auth**. **Sem PKCE**.
  Sem token em SecureStore. oEmbed scrape único, sem rede recorrente.
- **YouTube**: thumbnail derivada do ID, sem chamada de rede.
  Reprodução do vídeo está **fora de escopo** desta sprint
  (somente cover estático).
- **Áudio**: reusa pipeline da M06.5. Sem fallback de nuvem.
- **Modo trigger / negativo**: mídia continua opcional. UI mostra
  o `<MidiaPicker obrigatorio={false}>` mas não bloqueia o save.
- **Limite por registro**: 4 mídias máximo (cap interno; UI
  desabilita ao atingir).
- **Migração de schema**: o `.refine` adicionado em
  `diario_emocional` e `evento` é **breaking** para arquivos `.md`
  antigos sem `midia`. O default `.default([])` no campo cuida do
  parsing, mas o refine só dispara em **novos saves**. Arquivos
  existentes sem `midia` continuam legíveis (são reverenciados como
  sem mídia, sem mensagem de erro).

## 10. Decisões tomadas

- **Spotify oEmbed sem auth (final):** zero rede recorrente, sem
  token, sem PKCE. Cover fallback bonito quando offline:
  `<View>` com fundo `--green` 30% + ícone `Music` da
  `lucide-react-native` em `--green` 100%. Web API completa fica
  permanentemente rejeitada (contraria ADR-0007).
- **Cap de mídias configurável:** default 4 vindo de
  `useSettings.midia.capPorRegistro` (criado em M00.5; M15
  expõe slider 1-8). Revisão sob demanda real.
- **YouTube link externo** via `Linking.openURL` no detalhe; embed
  inline rejeitado (overhead de `react-native-youtube-iframe`
  desnecessário para o caso de uso).
- **Backward compat:** refine só vale para novos saves; arquivos
  legados sem `midia` ficam intocados (default `[]`).
- **M07.x bloqueia M11.5:** acoplamento desejado. Calendário só
  faz sentido com cover bonito de mídia.
- **`<MidiaPicker>` reusável** em sprints futuras que quiserem
  adicionar mídia opcional a outros schemas (ex: marcos
  manuais em M11).
