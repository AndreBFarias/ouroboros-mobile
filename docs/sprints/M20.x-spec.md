# Sprint M20.x — Validação Nível B do widget homescreen Android

```
DEPENDE:    M20 fechada (modulo nativo + bridge JS + helper TS)
            + EAS dev-client APK disponivel
BLOQUEIA:   nenhuma
ESTIMATIVA: 1-2h
PRIORIDADE: media (widget so funciona em runtime nativo Android)
STATUS:     [todo]
```

## 1. Contexto

M20 entregou módulo expo-widget-homescreen com 2 layouts (4x2 e
4x4), receiver Android (`OuroborosWidgetProvider.kt`), bridge JS
(`requireNativeModule`), e helper TS
(`atualizarWidgetHomescreen`). A revalidação 2026-05-03 marcou
M20 como INCONCLUSIVO no Gauntlet — Nível A+ valida apenas o
toggle "Widget na tela inicial" em `/settings`, mas o widget
nativo em si só renderiza no Android real.

## 2. Objetivo

Capturar 4 screenshots do widget renderizando em Android:

1. Widget 4x2 com humor médio do dia preenchido + 4 atalhos
   visíveis.
2. Widget 4x4 com tudo do 4x2 + heatmap mini 7 dias + frase do
   dia.
3. Widget após salvar humor via app: dados atualizados
   automaticamente (chamada de `atualizarWidgetHomescreen()`
   funcionando).
4. Toggle off em `/settings` desativa widget (some das opções de
   homescreen).

## 3. Procedimento

```bash
# 1. APK dev-client em mãos
ls builds/dev-client-*.apk
# (se ausente, solicitar build via EAS)

# 2. Instalar no emulador ouroboros-test
./scripts/start-emulator.sh
adb -s emulator-5554 install -r builds/dev-client-*.apk

# 3. Subir Metro
./run.sh --emulator

# 4. Adicionar widget na home do emulador (manual: long-press home,
#    arrastar widget Ouroboros).

# 5. Capturar screencaps via ADB
adb -s emulator-5554 shell screencap -p /sdcard/widget-4x2.png
adb -s emulator-5554 pull /sdcard/widget-4x2.png \
  docs/sprints/M20-screenshots/A-widget-4x2-emulador.png

# 6. Repetir para 4x4, atualizado pos-humor, e toggle off.
```
