## 10. Decisões tomadas

- **Tabs Expo Router file-based:** preferido sobre tabs custom em
  raiz para reaproveitar o roteamento nativo e simplificar deep
  links.
- **`useSettings` criado já com shape completo:** evita refactor
  cruzado quando M15 implementa UI; sprints opt-in já podem
  consumir os toggles existentes.
- **Boot hooks como array de funções:** alternativa simples a
  event emitter; cada sprint dona faz `BOOT_HOOKS.push(funcao)` no
  seu próprio módulo.
- **Tabs condicionais redirecionam para `/(tabs)/em-construcao`
  quando a sprint dona ainda não chegou:** evita hard-crash em
  navegação direta. Sprint dona substitui o redirect quando
  implementa.
- **EAS profiles fixos em 3 (development, preview, production):**
  cobre dev-client, preview interno e release de loja sem
  introduzir complexidade adicional.
