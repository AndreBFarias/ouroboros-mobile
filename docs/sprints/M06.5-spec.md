## 1. Objetivo

Adicionar o botão `Gravar áudio` inline na Tela 18 (acima do textarea
livre). **Press = grava** (haptic medium); **release = para**. Durante
a gravação mostra waveform ao vivo via `expo-av` `Audio.Recording`. No
release, dispara transcrição on-device com `@react-native-voice/voice`
(Voice/SpeechRecognizer Android) e preenche o textarea com o resultado.
O áudio bruto é salvo em `~/Protocolo-Ouroboros/assets/<timestamp>.m4a`
e o path relativo entra no campo `audio` do frontmatter
`diario_emocional`. Tudo on-device, zero rede (`ADR-0003`,
`ADR-0007`).

## 4. Restrições

- **Regra −1** (Anonimato): zero referência a IA, zero nomes reais
  hardcoded. Nem mesmo `'Voice'`, `'SpeechRecognizer'` na UI; nomes
  internos da lib ficam em comentários técnicos.
- Sem emojis em código, docs ou commits.
- Strings de UI em **Sentence case com acentuação completa PT-BR**
  (ADR-0013). Ex: `"Gravar áudio"`, `"Solte para transcrever."`,
  `"Sem permissão de microfone."`, `"Microfone indisponível agora."`,
  `"Transcrevendo…"`.
- `accessibilityLabel` sem acento (ex: `'botao gravar audio'`,
  `'parar gravacao'`).
- Comentários em código sem acento.
- Mensagens de commit sem acento.
- TypeScript strict.
- Imports via alias `@/*`.
- **Não tocar** em `src/components/ui/*` (M01).
- **Não tocar** em `src/lib/schemas/diario_emocional.ts` (campo
  `audio` já existe, é o suficiente).
- **Zero rede.** Toda transcrição é on-device. Se a lib quiser cair
  em fallback de nuvem, abortar e mostrar toast
  `"Microfone indisponível agora."`.
- **Áudio fica em `assets/`** dentro de `~/Protocolo-Ouroboros/`,
  apontado pelo path relativo `./assets/<file>.m4a` no frontmatter.
  Sync via Syncthing carrega o binário pro desktop.
- **Limite de gravação:** 60 segundos. Acima disso, parar
  automaticamente e exibir toast `"Áudio limitado a 60 segundos."`.
- **Não regressar** o comportamento de M06: salvar continua
  funcionando sem áudio (`audio: null`).

## 10. Decisões tomadas

- **dev-client primeiro:** rodar `npm run build:dev` (script
  criado em M00.5) e instalar APK no celular antes de começar a
  spec; sem isso, smoke não cobre a parte nativa.
- **Cloud build padrão:** ciclo nuvem (~15-25 min) é o default;
  build local (~10 min) opcional para quem tem ambiente Android
  configurado.
- **Whisper local fica para sprint M06.5.1 (sucessora) só se
  usuário relatar qualidade ruim do `SpeechRecognizer` nativo.**
  M06.5 entrega o caminho com Voice; sub-sprint só dispara sob
  demanda. Isso não é "fora de escopo" — é critério de evolução.
- **`.m4a` AAC qualidade alta:** preset
  `RECORDING_OPTIONS_PRESET_HIGH_QUALITY`. Conversão Opus 16kbps
  fica como otimização opcional em Settings (toggle
  `comprimirAudio` em sprint M15.1 se demanda surgir).
- **MicPermissionError UX:** primeira negação → toast simples;
  segunda → deep link `Linking.openSettings()`. Evita ser
  invasivo, mas garante recuperação.
