## Armadilhas descobertas (A32-A37)

### A32 — HyperOS bloqueia `adb install -r`

`INSTALL_FAILED_USER_RESTRICTED: Install canceled by user` mesmo
com "Install via USB" + "Depuração USB (Config segurança)" ATIVAS.
Popup HyperOS de confirmação nunca aparece para alguns APKs (filtragem
por assinatura sem garantia Mi).

**Fix canônico:**
```bash
adb push builds/<apk> /data/local/tmp/app.apk
adb shell pm install -r -t /data/local/tmp/app.apk
```

Funciona porque `pm install` roda como `shell` user (privileged) e
não passa pelo verificador HyperOS de origem do arquivo.

### A33 — `Header.tsx` `w-10` corta right slot

`<View className="w-10">` força 40dp no slot direito do Header,
cortando conteúdo maior (pill com ícone+label).

**Fix canônico:**
```tsx
// Antes: <View className="w-10 items-end justify-center">{right}</View>
// Depois:
<View style={{ minWidth: 40, alignItems: 'flex-end', justifyContent: 'center' }}>
  {right}
</View>
```

Cresce com conteúdo, mantém simetria com slot esquerdo (back 40dp).

### A34 — `Button.tsx` MotiView colapsa flex row complexo

MotiView interno do Button aplica `alignItems: center + justifyContent:
center` e isso colapsa filhos `<View flexDirection: row>` no New Arch
(release Android + web). Sintoma: só renderiza ícone, ignora Text irmão.

**Fix canônico:** para pills/CTAs com layout custom (ícone+label,
badge), usar `Pressable` direto inline (~30 linhas). Button.tsx
reservado para CTAs primários full-width com label string simples.

Exemplo: `BotaoRecap` em `app/index.tsx:35-78`.

### A35 — ADB tap usa coords físicas

Screencap pull vem como PNG ~540x1106 (2.0x scale-down do físico
1080x2400 no Xiaomi 2312DRAABG). ADB `input tap` usa coords físicas.

Calcular: `tap_x = screencap_x * physical_w / screencap_w` (= 2x no
2312DRAABG).

**Solução melhor que tap cego:**
```bash
adb shell uiautomator dump /sdcard/ui.xml
adb pull /sdcard/ui.xml /tmp/ui.xml
grep -oE 'text="[^"]*" .* bounds="[^"]*"' /tmp/ui.xml | head -20
```

Bounds vêm em coords físicas absolutas — usar diretamente.

### A36 — ANR em DevLauncherErrorActivity

App ANR (Application Not Responding) em `DevLauncherErrorActivity`
indica crash JS antes do React errboundary pegar — não aparece toast
de erro. Logcat só mostra transition do WindowManager + `Killing X:
user request after error`.

**Diagnóstico:**
```bash
PID=$(adb shell pidof com.ouroboros.mobile)
adb logcat -d --pid=$PID 2>&1 | grep -iE 'TypeError|Invariant|Cannot|fatal'
```

### A37 — Build EAS dev-client demora 10-25min

Build EAS dev-client APK demora 10-25min em fila Free Tier (compila
nativo + assina com keystore EAS).

**Critério para evitar rebuild:** se só código JS mudou (não plugins
Expo nem packages nativos), reusar `builds/dev-client-*.apk` local
e usar Metro local + `adb reverse tcp:8081 tcp:8081` é mais rápido.

## Protocolo de validação live no celular HyperOS

Sequência canônica testada na Onda Q:

```bash
# 1. Instalar dev-client APK (bypass HyperOS — A32)
adb push builds/dev-client-20260505-91710ab.apk /data/local/tmp/app.apk
adb shell pm install -r -t /data/local/tmp/app.apk

# 2. Conectar Metro local via USB
adb reverse tcp:8081 tcp:8081
nohup npx expo start --dev-client > /tmp/metro.log 2>&1 &

# 3. Aguardar Metro responder
until curl -s -o /dev/null -w '%{http_code}' http://localhost:8081 | grep -q 200; do sleep 2; done

# 4. Abrir app no Metro via deep link
adb shell am force-stop com.ouroboros.mobile
adb shell am start -W -a android.intent.action.VIEW \
  -d 'ouroboros://expo-development-client/?url=http%3A%2F%2Flocalhost%3A8081'

# 5. Aguardar bundle (~30s primeira vez)
# 6. Hot-reload automático em edits Edit/Write subsequentes
```

### Bloqueador A — Ciclo não persiste (root cause: autor mismatch)

**Confirmado runtime-real no celular real:** em config casal com
pessoa_a masculino + pessoa_b feminino, salvar registro gera arquivo
no disco com `autor: pessoa_b` (autorPadrao infere a pessoa feminina),
mas `app/ciclo/index.tsx` carregava com `pessoaAtiva = 'pessoa_a'`
(default da store) → filter exclui o registro → empty state apesar
do arquivo existir.

Prova literal:
```yaml
# /data/user/0/com.ouroboros.mobile/files/Ouroboros/markdown/ciclo-2026-05-12.md
tipo: ciclo_menstrual
data: 2026-05-12
autor: pessoa_b   # <- inferido por autorPadrao(casal, masc, fem)
data_inicio: 2026-05-12
fase: menstrual
sintomas: [colica]
```

**Fix** (app/ciclo/index.tsx): replicar a inferência de autor que
`registrar.tsx:95-98` já fazia, garantindo simetria save/load.

```ts
const autorListagem = useMemo(
  () => autorPadrao(tipoCompanhia, sexoA, sexoB) ?? pessoaAtiva,
  [tipoCompanhia, sexoA, sexoB, pessoaAtiva]
);
// e na chamada: listarRegistrosCiclo(vaultRoot, autorListagem, ...)
```

Validado após hot-reload: tela mostra "Dia 1 do ciclo / Duração média
detectada: 28 dias" + calendário com dia 12/05 destacado em vermelho
(fase menstrual) + lista "Últimos registros: Menstrual ter 12/05 ·
Cólica". E2E adicionado em
`tests/e2e/playwright/m-q8-ciclo-persistencia.e2e.ts`. 1802 testes
verde mantidos.
